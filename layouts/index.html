txn2 open source components.

txn2 is an open source organization for components developed by Craig Johnston, sponsored by Deasil Works, Inc. and Plexara. Includes command-line tools, Go libraries, and Model Context Protocol servers covering Kubernetes networking, host-file management, object storage, distributed SQL, and metadata catalogs.

All components are released on GitHub under the MIT or Apache 2.0 license.

§ 01 — command-line tools & libraries

kubefwd · txeh

FWD-001 · go · kubefwd.com ↗

kubefwd

Bulk port-forward Kubernetes services to a local network.

Walks one or more Kubernetes namespaces, opens a port-forward for every service, and writes hostname entries into /etc/hosts so service names resolve directly on the local machine. Written in Go.

~ — kubefwd 
$ kubefwd services -n the-project
INFO  Press [Ctrl-C] to stop forwarding.
INFO  'cat /etc/hosts' to see fwd entries.
  ✓ postgres                    127.1.27.1:5432
  ✓ redis                       127.1.27.2:6379
  ✓ elastic                     127.1.27.3:9200
  ✓ kafka-bootstrap             127.1.27.4:9092
  ✓ api-internal                127.1.27.5:8080
INFO  forwarding 23 services across 3 namespaces.

HST-002 · go · txeh.txn2.com ↗

txeh

Go library and command-line tool for managing /etc/hosts.

Adds, removes, and lists entries in the system hosts file. Idempotent operations and stable formatting. Used internally by kubefwd and as a standalone utility.

~ — txeh 
$ txeh add 127.0.0.1 dev.local api.dev.local
  ✓ added 2 hostnames

$ txeh list
  127.0.0.1     dev.local api.dev.local
  127.1.27.1    postgres elastic kafka
  ::1           localhost

$ txeh remove dev.local
  ✓ removed 1 hostname
visit txeh.txn2.com → source ↗

§ 02 — model context protocol servers

MCP servers for the data stack.

Model Context Protocol servers exposing object storage, distributed SQL engines, and metadata catalogs to MCP-capable clients.

{{/* big card */}}

platform

mcp-data-platform

MCP server providing integrated access to S3-compatible object storage, the Trino distributed SQL query engine, and the DataHub metadata platform from a single endpoint.

  • browse, query, and inspect across S3, Trino, and DataHub
  • persistent memory and insight capture across sessions
  • artifact storage and prompt management
  • single-server alternative to running each MCP integration separately
